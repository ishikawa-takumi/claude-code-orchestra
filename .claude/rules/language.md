# Language Rules

## Thinking and Reasoning

- **Always think and reason in English**
- Internal analysis, planning, and problem-solving should be in English
- Code comments, variable names, function names, and docstrings should be in English

## User Communication

- **Always respond to users in English**
- Explanations, questions, and status updates should be in English
- Error messages shown to users should be in English

## Code

- All code should be written in English:
  - Variable names: `user_count`, not `<non_english_name>`
  - Function names: `calculate_total()`, not `<non_english_name>()`
  - Comments: `# Check if valid`, not `<non_english_comment>`
  - Docstrings: English descriptions

## Documentation

- Technical documentation: English
- User-facing documentation (README, etc.): English
