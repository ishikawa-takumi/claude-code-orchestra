$ARGUMENTS の実装計画を立ててください．

## 計画プロセス

### 1. 要件分析

まず以下を明確にする：

- **目的**: 何を達成したいか
- **スコープ**: 何を含み，何を含まないか
- **制約**: 技術的制約，時間的制約，依存関係

### 2. 現状調査

コードベースを調査：

```
- 関連する既存コード
- 影響を受けるファイル
- 使用するライブラリ・パターン
- テストの有無
```

### 3. 実装ステップの分解

タスクを小さなステップに分解：

1. 各ステップは独立してテスト可能
2. 依存関係の順序を考慮
3. リスクの高いステップを先に

### 4. 出力フォーマット

```markdown
## 実装計画: {タイトル}

### 目的
{1-2文で}

### 影響範囲
- 新規ファイル: {リスト}
- 変更ファイル: {リスト}
- 依存ライブラリ: {リスト}

### 実装ステップ

#### Step 1: {タイトル}
- [ ] {具体的なタスク}
- [ ] {具体的なタスク}
**検証**: {このステップの完了条件}

#### Step 2: {タイトル}
...

### リスク・注意点
- {潜在的な問題と対策}

### 未解決の質問
- {実装前に確認が必要な点}
```

## 注意事項

- 計画は実行可能な粒度に
- 各ステップに検証方法を含める
- 不明点は計画段階で質問する
- 過度に詳細にしすぎない（実装時に調整）
